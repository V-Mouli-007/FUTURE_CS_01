# Security Policy

## Project Status
This repository contains a static vulnerability assessment report and methodology documentation. As this is not an active software application, there are no "supported versions" of code to maintain.

## Educational Purpose & Scope
The contents of this repository are strictly for educational and portfolio demonstration purposes. The vulnerabilities documented herein were discovered on an authorized, safely contained internal testing environment (or explicitly permitted educational test domains). 

⚠️ **Do not attempt to replicate these findings on production systems or any domains where you do not have explicit, written authorization to test.**

## Reporting a Vulnerability
If you discover a legitimate security issue within the actual structure of this repository (e.g., accidentally leaked personal credentials, exposed API keys, or sensitive local network data in the documentation), please report it immediately.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please practice responsible disclosure by sending an email directly to:
📧 **moulieswar007@gmail.com**

I will acknowledge the receipt of your vulnerability report within 48 hours and work to remediate the issue (e.g., scrubbing the git history of the exposed secret) as quickly as possible.
